# バージョン管理方針

このアプリの公開版は、Gitの注釈付きタグで固定する。

## バージョン番号

`vメジャー.マイナー.パッチ` の形式を使う。

- パッチ（例: `v1.0.1`）: 軽微な不具合修正、表示調整
- マイナー（例: `v1.1.0`）: 後方互換性のある機能追加、まとまった改善
- メジャー（例: `v2.0.0`）: 大きな仕様変更、互換性のない変更

法務局データの自動更新だけでは、アプリのバージョン番号を上げない。

## 改修の流れ

1. `main` の最新版から改修用ブランチを作る。
2. 改修用ブランチで変更し、動作確認する。
3. 確認済みの変更を `main` に反映する。
4. `VERSION` を更新し、同じ番号の注釈付きタグを付ける。
5. タグをGitHubへpushし、変更内容を記録する。

## 以前の版へ戻すとき

公開履歴を壊さないため、`main` を強制的に過去へ戻さない。戻したいタグと現在版の差分を打ち消すコミットを作り、そのコミットを `main` に反映して再公開する。

タグは永久保存し、削除・付け替えをしない。
